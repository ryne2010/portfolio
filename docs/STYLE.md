# STYLE.md

## Writing style

- Clear, technical, and credible.
- Senior-level tone without hype.
- Prefer specific architecture statements over marketing language.
- Never imply accomplishments that are not encoded in content files.

## Code style

- Prefer explicit names over shorthand.
- Keep route definitions small and legible.
- Keep helper functions pure where possible.
- Use utility classes for layout and spacing; avoid bespoke CSS unless it improves clarity.

## Content style

- Placeholder content should be marked openly.
- Portfolio copy should explain decisions, constraints, and tradeoffs.
- Project pages should emphasize architecture, operations, and validation over feature lists.

## Formatting

Formatting is automated through Biome and the Python harness entrypoints.
